# Agent Instructions

This directory formalizes agent-facing working contracts for the public theory repository.

## Role
These files do not replace the theory source tree.
They formalize how the agent must work on the tree:
- what artifact sequence to create
- how to separate theory, derivation, code, simulations, and results
- how to maintain cross-references
- how to track open questions and epistemic status
- how to preserve relational and publication hygiene

## Mandatory use
Any substantial addition to the repository should follow the patterns defined here before being treated as stable or merge-ready.

## Core files
- `FOUNDATIONAL_WORK_PATTERN.md` — canonical multi-artifact work pattern
- `RELATIONAL_CONTRACTS.md` — formalized interaction and epistemic contracts
- `INDEXING_AND_CROSSREF_PROTOCOL.md` — required indexing and cross-reference discipline
- `OPEN_QUESTIONS_PROTOCOL.md` — how to record unresolved questions and answered questions
