# Constants Index

This index organizes the constants layer of the repository.

## Role
Constants here are not mere numbers. Each constant must be a registered object with:
- a symbol
- a formal role
- a derivation path
- executable representation
- tests
- epistemic status
- provenance

## Current canonical objects
Declared in `constants/registry.yaml`:
- I0
- Lambda0
- tau_i
- kappa

## Current state
The registry and folders exist, but the inner content is still mostly placeholder material.
That means constants are declared but not yet fully canonized.

## Required completion criteria for each constant
1. canonical README / role statement
2. derivation document
3. code representation
4. tests
5. interpretation limits
6. registry links to derivations/code/tests/status

## Rule
A constant may not be treated as derived if its numerical value is injected directly without declared dependency path.
