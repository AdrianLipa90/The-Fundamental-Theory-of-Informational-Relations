## First AGENT4 interpretation
For AGENT4 purposes, the current working hypothesis is:
- `systems/CIEL_FOUNDATIONS/` is native-first-principles territory,
- `systems/CIEL_OMEGA_COMPLETE_SYSTEM/` is to be treated first as a coupling sector rather than automatically as a scope contradiction,
- reference/vocabulary/memory trees require explicit status marking,
- reports/comparison folders are likely provenance-heavy and must not be confused with canonical formal objects.

## First indexing consequences
The next inventory pass should explicitly distinguish:
- native formal objects,
- coupled Omega-facing objects,
- imported objects,
- mirrored/reference objects,
- provenance/report objects,
- placeholders,
- export-ready objects.
